# MVP Pivot — 취향 결정화 에이전트로의 전환

**일시**: 2026-02-19 (인터뷰 후 방향 확정)
**배경**: 예선 마감 D-1, 데모 방향 피봇

---

## 등장 배경: 기존 기능의 문제점

초기 설계에서는 "커버곡 감지 + 플랫폼 링크 수집"이 AI의 핵심 역할이었다.
인터뷰 과정에서 세 가지 근본적 문제가 확인됐다:

### 1. AI 가시성 없음

AI가 무언가를 하고 있다는 것이 UI에서 **전혀 보이지 않았다.**
- 에이전트가 실행되는 동안 `"analyzing..."` 텍스트 하나가 전부
- 심사위원 입장에서 "AI를 어디에 썼나요?" 질문에 답하기 어려움
- 배점 항목 중 **AI 활용도 5점**을 가장 어필하기 어려운 구조

### 2. 플랫폼 링크 기능 신뢰도

OpenAI 웹 검색으로 수집한 Spotify/Apple Music 링크가 **부정확한 경우**가 발생했다.
- 동명의 다른 곡 링크를 반환하는 케이스
- 플랫폼별 URL 패턴이 달라 검증이 어려움
- 비즈니스 서사에서도 부차적인 기능에 해당

### 3. 취향 데이터 없음

결정적 문제: "내 취향을 기억한다"는 핵심 서사를 뒷받침하는 **데이터가 쌓이지 않았다.**
- 트랙마다 아티스트 컨텍스트는 있지만 장르/무드 분류가 없음
- 누적 데이터 없이 취향 대시보드 불가
- 5whys 분석에서 도출한 가장 중요한 가치("취향 결정화")가 구현되지 않음

---

## 원리: 5 Whys → 취향 결정화 → Genre+Mood → 대시보드

```
5 Whys 핵심 인사이트
  ↓
"플리만 듣다 취향이 휘발됨"
  ↓
해결책: 플리를 분해해서 각 곡에 의미를 붙인다
  ↓
의미 = Genre + Mood (AI가 분류)
  ↓
누적 → 취향 대시보드 ("내가 이렇게 Lo-Fi를 좋아했구나")
  ↓
장기: 내 취향을 아는 개인 AI 에이전트
```

### AI가 하는 일 (변경 후)

| Before | After |
|--------|-------|
| 커버곡 여부 감지 | **Genre 분류** (K-Pop, Indie Rock, Lo-Fi Hip-Hop…) |
| 원곡 아티스트 찾기 | **Mood 분류** (Chill, Energetic, Melancholic…) |
| 플랫폼 링크 수집 | 아티스트 컨텍스트 + 곡 설명 |

OpenAI Responses API + `web_search_preview` 툴로 웹 검색을 통한 정확한 분류.
AI 작업 중 렌더러에 실시간 스텝 푸시 → **AI 가시성 확보**.

## 제거된 기능과 근거

### Albums / Artists 뷰 제거

- **근거**: URL 임포트 중심 제품에서 앨범/아티스트 계층 구조는 의미가 없다
- 대부분의 임포트 트랙은 `albumTitle = null`이어서 Albums 뷰가 비어있음
- 사이드바 노이즈 감소 → **더 집중된 UX**

### Obsidian 연동 제거

- **근거**: 타겟(딥워크 지식 노동자)에게는 매력적이나 **데모 3분 내에 설명하기 불가능**
- 심사 배점 구조(사업성 10점)에서 Obsidian 연동은 기여도 낮음
- 코드 복잡도 증가 vs 심사 가치 불균형

### 플랫폼 링크 제거

- **근거**: 신뢰도 문제 (위 참고) + 새 방향(Genre/Mood)과 기능적으로 다른 맥락
- "다른 플랫폼에서 찾아줌" → "내 취향을 알아줌"으로 가치 전환

### 로컬 폴더 스캔 제거

- **근거**: URL 임포트가 핵심 행동이며, 폴더 스캔은 데모에서 보여주기 어려움
- 해커톤 데모 흐름: "URL 입력 → AI 분류 → 취향 확인" — 단일 행동에 집중

## 데모 시나리오 (3분)

### 0:00 – 0:20 | 오프닝 훅

> "유튜브 플레이리스트에서 좋아하는 노래를 발견했는데, 나중에 다시 못 찾은 적 있죠?
> 그리고 저는 최근 제가 어떤 음악을 왜 좋아하는지 모르겠더라고요.
> 플리만 틀고 있으니까요."

공감 유도 → 잠재적 니즈를 인식 니즈로 전환.

### 0:20 – 1:00 | 임포트 데모

1. YouTube URL 입력
2. 다운로드 프로그레스바 실시간 확인
3. **AI 분석 단계 UI**: "🔍 곡 정보 검색 중...", "🎵 장르/무드 분류 중..."
4. 트랙 카드에 `Lo-Fi Hip-Hop · Chill` 태그 등장

**핵심 포인트**: AI가 일하는 모습을 실시간으로 보여준다 → AI 활용도 어필.

### 1:00 – 1:30 | 트랙 카드 & 재생

- 태그(Genre · Mood pill)가 붙은 트랙 카드 강조
- 더블클릭으로 재생 → PlayerBar에서 재생 확인

### 1:30 – 2:30 | 취향 대시보드

- 사이드바 `My Taste` 탭 클릭
- **Genre 분포**: "Lo-Fi Hip-Hop 35%, Indie Rock 25%..."
- **Mood 분포**: "Chill 50%, Energetic 30%..."
- "여러분이 어떤 음악을 좋아하는지, 이 앱이 기억합니다."

### 2:30 – 3:00 | 비즈니스 모델

- Settings에서 크레딧 잔액 확인
- "기본 10 크레딧 무료, AI 분석 1회 = 1 크레딧"
- 북극성: **내 취향을 아는 개인 AI 에이전트**

## 크레딧 수익 모델

```
Free Tier               Paid Credits
────────────            ──────────────────────
가입 시 10 크레딧        크레딧 팩 구매
URL 임포트 (무제한)      AI 분석 횟수 무제한 사용
AI 분석 10회            취향 레포트 (월간)
기본 대시보드            심화 취향 분석
```

### 왜 구독이 아닌 크레딧인가?

- 취향 AI의 가치는 **사용량에 비례**한다 — 많이 분석할수록 더 나은 추천
- 구독은 "쓰든 안 쓰든 내는 돈" 심리적 장벽 높음
- 크레딧은 **가치 발생 시점에 지불** — 유저가 AI 효과를 체감한 후 결제
- SaaS → **Usage-based AI** 모델로 진화하는 시장 트렌드에 부합

## 기술 변경 사항 요약

| 영역 | 변경 전 | 변경 후 |
|------|---------|---------|
| AI 결과 | `isCover`, `originalArtist`, `platformLinks` | `genre`, `mood`, `summary` |
| IPC 이벤트 | `import:enriched` (결과만 푸시) | 단계별 스텝 푸시 + 결과 |
| 사이드바 | Songs / Albums / Artists | Songs / My Taste |
| 트랙 카드 | 제목 + 아티스트만 | 제목 + 아티스트 + Genre·Mood 태그 |
| TrackMetadataPanel | 플랫폼 링크 목록 | Genre·Mood·Summary 표시 |
