# 데모 대본 (5 Whys 기반, 1분)

## 0) 오프닝 훅 (0:00 ~ 0:12)
- 멘트: **"유튜브 플레이리스트에서 좋은 노래를 발견했는데, 나중에 다시 못 찾은 적 있죠?"**
- 화면: 앱 홈 → 사이드바/목록/플레이어 핵심 UI 1초 훑기
- 연결 문장: **“이게 왜 생기는지, 5 Whys로 짧게 정리해보겠습니다.”**

## 1) Why #1: 왜 로컬인가? (0:12 ~ 0:20)
- 문제: 브라우저 이동이 곧 컨텍스트 스위칭 → 집중이 깨짐
- 동작: `내 음악` 화면을 보여주고, 곧바로 트랙 목록으로 이동
- 멘트: **“브라우저를 뜯어보느라 집중이 깨지는 순간이 제일 큰 손실입니다.”**

## 2) Why #2: 왜 기존 대안이 안 되나? (0:20 ~ 0:30)
- 문제 포인트: 삭제/권한 의존/폴더 메타 부재
- 동작: `미분류` / `태그 없음` 필터를 눌러 “현재 데이터는 분류 기준이 없으면 무너지는 구조”를 보여줌
- 멘트: **“스트리밍은 사라질 수 있고, 폴더만으론 태그·플랫폼 컨텍스트가 잘 안 남습니다.”**

## 3) Why #3: 왜 AI가 필요한가? (0:30 ~ 0:42)
- 문제 포인트: 플리 기반 청취로 취향이 휘발
- 동작: `+ 노래 추가` 클릭, 예시 URL 입력 후 임포트 시작 → 진행 상태 바(다운로드/메타/AI 검색/분류) 확인
- 멘트: **“한 곡의 가수, 원곡 여부, 출처 링크를 AI가 붙여서 내가 나중에도 다시 이해할 수 있게 만듭니다.”**

## 4) Why #4: 누가 고통받나? (0:42 ~ 0:52)
- 문제 포인트: 딥워크 지식 노동자(우리 대상)
- 동작: 트랙 카드 클릭 후 사이드 패널에서 즐겨찾기/태그 조작(로컬 메타 편집)
- 멘트: **“집중 몰입 중에 가장 큰 병목은 ‘좋았던 곡을 다시 기억 못 하는 순간’이죠.”**

## 5) Why #5 + 비즈니스 모델 (0:52 ~ 1:00)
- 문제 포인트: 취향 에이전트 필요성 + 가치 포인트
- 동작: `설정` 열기 후 Light/Dark/System 토글 빠르게 보여주기, `취향`으로 이동
- 멘트: **“로컬 재생은 기본값이고, AI 분석은 크레딧 기반으로 내 취향 에이전트로 확장됩니다. 브라우저 바깥에서도 내 취향이 사라지지 않습니다.”**

## 마무리 한 줄
- **"URL 하나로 노래를 소유하고, AI가 내 취향을 기억하게 만드는 로컬 뮤직 에이전트가 핵심입니다."**

### 촬영 체크 (1분 버전)
- 각 씬 뒤 0.3~0.5초 휴지기
- 모달 오픈/클로즈는 빠르게 연달아 누르지 않기
- 재생 버튼은 실제 오디오 유효 시에만 눌러 실패 화면이 길어지는 걸 피함
